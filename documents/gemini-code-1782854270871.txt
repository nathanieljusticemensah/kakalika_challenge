# GDSS-PSInno AgriTech Innovation Challenge: System Requirements & Implementation Plan

## 1. Project Overview
**Objective:** Develop a Farmer-to-Buyer Digital Marketplace to reduce Ghana's post-harvest vegetable losses by connecting smallholder farmers, buyers, and transport providers.
**Target Output:** A functional Minimum Viable Product (MVP) with frontend UI, backend database, and a live demo of the end-to-end workflow by July 10, 2026.

## 3. Requirement Analysis

### A. Core Marketplace Features (The "Must-Haves")
**1. Farmer Module**
*   *Registration & Profiles:* Secure sign-up with farm location data.
*   *Inventory Management:* Upload produce (images, crop type, quantity in kg/baskets, price, location).
*   *Order Management:* Receive notifications, chat with buyers, and request transport.

**2. Buyer Module**
*   *Search & Discovery:* Filter produce by category (tomatoes, peppers, etc.), maximum price, radius/distance, and volume.
*   *Transaction Flow:* Direct ordering, Mobile Money (MoMo) checkout.
*   *Order Tracking:* History, receipts, and delivery status monitoring.

**3. Logistics & Transport Module (Frederick's Domain)**
*   *Driver Profiles:* Vehicle type registration (tricycle, van, truck) and load capacity.
*   *Job Board/Push Notifications:* Receive delivery requests based on proximity to the farmer.
*   *Trip Management:* Accept/reject trips, status updates (Arrived at Farm, In Transit, Delivered).

### B. Competitive Edge Features (The "Stand-Outs" / Bonuses)
**1. Smart Matching & Recommendation System (Kojo)**
*   *AI Pricing Estimation:* An XGBoost regression model to estimate delivery costs based on distance, road conditions, and payload size.
*   *Proximity Matching:* Automatically ping the 3 closest available drivers when a buyer makes a purchase.

**2. Offline & Low-Bandwidth Access**
*   *USSD/SMS Integration:* Allow farmers without smartphones to dial a shortcode (e.g., *713#) to check market prices or list a new batch of harvested vegetables.

## 4. Technical Implementation & Architecture ("How to Achieve It")

### Recommended Tech Stack
*   **Frontend (User Interfaces):** Flutter (for a unified iOS/Android mobile app) OR React/Next.js (for a highly responsive web application). 
*   **Backend (Core Logic):** Node.js with Express OR Python (FastAPI/Django) – Python is highly recommended to easily serve Kojo's machine learning models.
*   **Database:** PostgreSQL (Ideal for relational data like user roles, orders, and financial transactions) + PostGIS extension for spatial/location queries.
*   **Mapping/Geolocation:** Mapbox API or Google Maps API (Frederick will use this for route plotting and distance calculation).
*   **Payments:** Paystack API or Hubtel API for seamless Ghana MoMo integration.
*   **Offline/USSD:** Africa's Talking API (provides easy SMS and USSD infrastructure for Africa).

### Key Architectural Workflows

**1. The Data Pipeline (How ML integrates)**
*   *Data Collection:* Afia curates dummy dataset of Tarkwa farmers and local drivers.
*   *Training:* Kojo trains an XGBoost model on historical distance-to-cost ratios.
*   *Deployment:* The Python backend exposes an endpoint `/api/estimate-shipping` that Frederick's frontend calls when a buyer views their cart.

**2. The Logistics Engine (Frederick's Implementation Steps)**
*   *Step 1 (Location):* Capture GPS coordinates of the Farmer and Buyer during registration/checkout.
*   *Step 2 (Routing):* Use Mapbox API to calculate the driving distance between Point A and Point B.
*   *Step 3 (Matching):* Query the database for drivers with a status of "Available" within a 10km radius of the Farmer.
*   *Step 4 (Tracking):* Implement WebSockets (e.g., Socket.io) so that when a driver clicks "Picked Up," the buyer's app instantly updates to "In Transit" without refreshing the page.

**3. The Payment Flow (Daniel's Implementation Steps)**
*   Buyer selects produce and clicks "Pay".
*   Backend initiates a Paystack MoMo prompt to the buyer's phone.
*   Upon successful PIN entry, the funds are held in an escrow-like state on the platform.
*   Only when the driver marks "Delivered" and the buyer confirms, do the funds disburse to the Farmer and Driver wallets.
